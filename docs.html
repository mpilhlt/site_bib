The dataset, which is used in this app, consists of 1.300 thematic books. It represents an export of an EndNote database in the bib format, transformed into a data frame. This convertion allows to present a catalogue with textual units as a data table for further manipulations in R.

Therefore, the basic step of the development was a bib. To data frame transformation. For these needs CRAN has a package bib2df, which worked perfectly for this dataset.

Motivation

The aim of this project is to develop the application for the library catalogues with complext smart functions and graphic visualization. The application must be multi-platform, lightweight, fast and applicable for any thematic catalogue, regardless the subject. Therefore, in the future, the code of this app could be used for other specified libraries with relatively small dataset (about 10.000). However we are working on the performance improvement for the larger datasets.

The main task of this project is visualization of the Congregation of the Council catalog with multiple search and visualization options.

Instruments

The main language of this app is R. It’s set up because of simplicity of operations with data tables and rich graphic visualization via ggplot2. Active elements in this application are based on shinyjs or JavaScript itself. A visual theme is bootstrap 4 ‘Sandstone’, available as an option of the shinythemes package and bootstrap 4 css

All R and html files are compiled with shiny library.

Pages

Smart Search

This page provides the search manipulations with the data. It consists of the sidebar layout with filters and data table selections.

The filters are divided by the general search and advanced search

Search Data

Type names, words or numbers you wish to find

“Everything in the database” provides a search among all titles, including dates, names or any other elements. The input can include character or numeric symbols and will give all possible results. This element is based on the proxy data table - it is needed for updating already existed data table instead of additional rendering. In this case, in the server side we use observeEvent() and function updateSearch() from DT:

ui 
textInput("search", "Everything in the database")

server

DTproxy <- dataTableProxy("Table")
observeEvent(input$search,{
updateSearch(DTproxy, keywords = list(global = input$search, columns = NULL))
})

Advanced Search

You can also combine several categories for the advanced search. Since the basic search does not reflect the structure of the Congregation, but mechanically show all possible results, we created an advanced search option, which includes several filters.

The slider input allows to cut the possible dataset by the year of publication. Since the slider can operate only with the natural numbers (i.e. 2000, but not a range 2000-2001), all selection is made by the first year in the case of double numbers. However in the data table there is an original EndNote column ‘YEAR’, which provides the original date of publication. In this case we should take the minimum min() and maximum max() possible numbers. Then in the server part this element should be included inside a reactive subset, which combines keywords and slider. The code for this part:

ui
h4("Time period"),
sliderInput("slider", label=("Choose particular years of the bibliography"),
min=min(dataset$year), max=max(dataset$year),
value=c(min(dataset$year),max(dataset$year)), sep = "")
server side 
...
# the dataset is dynamic, therefore we need a reactive function

dsub_dataset <- reactive({
key_search <- paste0(c('xxx',c(input$type_literature)), collapse = "|")
key_search <- gsub(",","|",key_search)
dataset[dataset$year >= input$slider[1] & dataset$year <= input$slider[2] & grepl(key_search, KEYWORDS)]
